Dataflöde: Webview → Insight Card

Hela kedjan från att användaren besöker en sida till att en insight sparas i plattformen.

1
Användaren navigerar i webview
Besöker t.ex. en konkurrents produktsida eller en SERP. Webview avfyrar did-navigate event.
webview.addEventListener('did-navigate', ...)
2
Content Extraction
Renderer kör executeJavaScript i webviewen — hämtar title, meta, headings, schema, OG-tags och brödtext. Samma mönster som befintliga get_seo_analysis.
webview.executeJavaScript(extractionScript)
3
VERA-analys via Claude API
Extracted content + klientkontext skickas till Anthropic API (redan kopplad i main.js). System prompt: "Klassificera intent, identifiera gaps, ge actionable insight." Svaret parsas och visas i sidopanelen.
ipcRenderer → main.js → api.anthropic.com
4
Användaren trycker "Generate insight"
VERA-insikten, intent-klassificeringen och sidans data paketeras som ett Insight Card-objekt med clientId från aktiv klientkontext.
{ type: 'insight', clientId, url, intent, vera_analysis, raw_data }
5
POST till SEOZ-plattformen
Browsern postar till seoz.se/api/insights via befintliga SeozApiClient. Ingen lokal IPC-brygga behövs — API:t är redan kopplat.
SeozApiClient.post('/api/insights', insightCard)
6
Supabase → Dashboard
Next.js-appen sparar till Supabase. Insight Card dyker upp i dashboarden, tillgänglig för OTTO (brief generation) och export till slides.
Supabase: insight_cards table → realtime subscription

Alternativt flöde: "Create campaign idea"

4b
Användaren trycker "Create campaign idea"
Istället för att spara en insight-card triggas Brief Generator-flödet direkt. VERA-analysen + sidans data skickas till OTTO.
SeozApiClient.post('/api/briefs', { clientId, source_insight, ... })
5b
OTTO genererar brief
Brief Orchestrator (5-fas pipeline) kör: insight → audience + message + tonality + 3 kampanjidéer. Saga Signalström-agenten skapar briefen.
brief-orchestrator.ts → 5 phases → Supabase
Kommunikationsarkitektur

Ingen lokal WebSocket-brygga behövs. Browsern har redan en HTTP-koppling till seoz.se/api. Nya endpoints + klientkontext i electron-store räcker.

Electron Browser
Renderer (Panel UI)
Insight Panel, generate-knappar, klient-dropdown. Triggar actions.
window.seoz IPC
Befintlig contextBridge. Utökas med: setActiveClient, getActiveClient, postInsight, postBrief.
Main Process
Hanterar IPC, lagrar clientId i electron-store, proxy:ar requests med auth header.
MCP Server (:19532)
Befintlig. Lägger till clientId i alla tool-responses. Claude Code/Desktop kan fråga vilken klient som är aktiv.
SeozApiClient
HTTPS + Bearer token
JSON responses
Auth-delning
JWT vid uppstart
SEOZ Platform
GET /api/clients
Finns. Browsern hämtar klientlista vid uppstart → populerar dropdown.
POST /api/insights ✦
Ny. Tar emot { clientId, url, intent, vera_analysis, raw_data }. Sparar som Insight Card.
POST /api/briefs ✦
Ny. Triggar Brief Generator. Tar insight + clientId → OTTO pipeline.
Supabase
insight_cards, briefs, client_signals. Realtime subscriptions för dashboard-uppdateringar.

Nya IPC-handlers i main.js

setActiveClient(clientId, name)
Sparar till electron-store. Emittar 'client-changed' event till renderer.
getActiveClient()
Returnerar { clientId, name } från store. Används vid uppstart och av MCP-server.
postInsight(data)
Proxy: lägger till auth header + clientId, postar till /api/insights.
postBrief(data)
Proxy: lägger till auth header + clientId, postar till /api/briefs. Returnerar briefId.
Implementationsordning

Ordnad efter: vad låser upp mest? Klientkontext först — allt annat beror på den.

1
Klientkontext i browsern
Dropdown i toolbar för att välja aktiv klient. Sparar clientId i electron-store. Alla API-anrop inkluderar clientId. Hämtar klientlista från befintligt GET /api/clients.
2–3 dagar Låser upp allt main.js, preload.js, index.html, api.js
2
API-endpoints: /api/insights + /api/briefs
Två nya Next.js API-routes i SEOZ-plattformen. Tar emot data från browsern, sparar till Supabase. Insight Cards-tabell i databasen.
2–3 dagar Browser → Platform-brygga src/app/api/insights/, src/app/api/briefs/
3
Sidopanel: VERA + Intent + Generate-knappar
Utöka .seoz-panel med tre sektioner. VERA triggas på did-navigate via befintlig Claude API-integration. Intent-klassificering är en del av VERA-prompten.
1 vecka Användarupplevelsen Kräver steg 1 + 2 index.html → bryt ut panel.js
4
Auth-delning
Next.js-appen skickar JWT till browsern vid uppstart (deep link eller clipboard-handoff). Browsern lagrar token i electron-store per profil. SeozApiClient skickar som Bearer header.
1–2 dagar Säkerhet & UX main.js, profile-manager.js, api.js
5
Search Everywhere Mode
Dedikerad vy med 4 parallella webviews. Scraping-scripts per källa. Aggregerad 3-kolumns-presentation. Mest ambitiösa delen.
2–3 veckor Differentiering Kräver steg 1–3 index.html → search-everywhere.js (ny modul)
6
Refaktor: bryt ut renderer-moduler
7700 rader index.html → moduler: panel.js, tab-manager.js, search-everywhere.js, mcp-bridge.js. Inte kritiskt nu men nödvändigt innan Search Everywhere.
3–4 dagar Maintainability index.html → src/renderer/*.js
